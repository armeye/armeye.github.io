Anspruchsdenken

Lennart Poettering ist im Moment eine der meist gehassten Personen in der Opensource Welt. Es scheint fast so, als ob er Linux ganz allein in den Angrund treiben würde. Was natürlich Dummtüch ist, wie der Norddeutsche sagen würde. Im Gegensatz zu den meisten seiner Kritiker hat er allerdings einen offensichtlichen Vorteil. Nämlich ein ganzes Repository voll Code, das ein von ihm gesehenes Problem lösen soll. Damit hat er erst mal gut vorgelegt, nur scheint sonst keiner nachzuziehen oder Geld in eine andere Lösung zu stecken. Opensource sollte zu dieser Einstellung zurückkehren: Code it, pay for it or shut yer piehole!

Man soll mich nicht falsch verstehen, ich halte nicht viel von Systemd. Das ist im besten Fall ein überkompliziertes Ding von fragwürdiger Codequalität um Probleme zu lösen, die man ohne es nicht hätte. Das Problem ist allerdings systemisch. UNIX Desktops leiden unter den selben Problemen. Warum ein Shellscript verwenden oder ein gut durchdachtes c/go/rust Programm schreiben, das ein Problem löst und über das man eine GUI klebt; wenn man das Ganze in freundlicheren Sprachen wie Js, Python, c++, Java oder Vala mit einem Haufen DSOs schreiben kann. Alle großen DEs sind deshalb quasi verkrüppelt. Und Systemd und Pulseaudio sind nur ein weiteres Symptom. Naja egal.

Come on, mate. Don't be a cunt. Show me the code -- Billy Butcher, quasi...
Lennart hat jedenfalls Code vorgelegt, was man von den ganzen Schreihälsen, die ihm die Pest an den Hals wünschen nicht sagen kann. Und genau da liegt jetzt das Problem, die Leute haben eine Anspruchshaltung gegenüber den Programmierern entwickelt, die ihnen die Software unentgeldlich zur Verfügung stellen. Entweder man legt was Besseres vor oder sucht sich nen neuen Spielplatz. Slackware, CRUX, LFS oder eins von den BSDs. Wer unbedingt Geld ausgeben will, kann ja Codebounties ausgeben, es ist erstaunlich, wie Geld Leute motiviert. Gibt genug andere Systeme.
