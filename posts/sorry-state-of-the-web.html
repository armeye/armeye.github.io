When Tim Bernes-Lee descended the mountain in 1991 with two stone tablets inscribed with html and http all things were well. After that, stuff started to go downhill. People began to bolt on many new additions like css or took it in ill devised directions like xhtml. Css grew version numbers, browser vendors added their own spin on the whole thing, some demented minds introduced dynamic web pages and finally Netscape took the last few pages from the Necronomicon and implemented them as JavaScript. The browser vendors followed them willingly on their path into madness. Today, a modern browser is more complex than most operating systems, consumes memory like a black hole and contains about a dozen parsers and jit compilers. IHMO web browsers are probably turing complete and fullfill Greenspun's tenth rule and Zawinski's law.
Now, there are a few bits which actually improved on the god given standard. I see html as a way to link structured text together and present it in a pleasant way. Downloadable fonts and the video/audio tags are among them.
So, here are my tips for building a user friendly website
  • Css is like Carolina Reaper, best used in very small doses
  • Avoid JS as much as possible, it's the path to hell. Try to get by with static html tags.
  • No Flash ever
  • Black text on a white background.
  • Make your pages less dynamic. Let scripts build your pages static.
  • Test your websites with lynx. If they don't look nice in lynx, you fucked up.
  • No autoplay for videos or sound. Come on, what is wrong with you?!
